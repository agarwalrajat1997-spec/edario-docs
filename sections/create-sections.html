Overview What Create Sections does before scheduling
In Progress

Create Sections is where you and the system collaborate to finalize what sections exist next year. You will (1) review section counts and assignments, (2) create and apply teams/labels to sections, and then (3) assign teams/labels to students for grouping logic.

Step 6 sub-pages. Create Sections includes: Section Review, Create Labels, and Student Labels.
Recommended workflow. Review sections first (counts + teachers) → then create teams/labels → finally assign labels to students.
Where to find it

Scheduling Wizard → Step 6: Create Sections

Sub-pages are shown in the left navigation within Step 6.

Completion standard
  • Section Review completed for each department (as applicable).
  • Any needed Teams/Labels created and applied to relevant sections.
  • Student Labels applied where grouping is needed.

Section Review Confirm section counts, teachers, and demand vs supply
In Progress

Section Review is where you manage the sections you are running next year: confirm section counts per course, assign teacher(s) to sections, add/adjust sections manually, and rerun auto-sectioning if needed.

2.1 — Department list screen

On the Section Review landing page, you’ll see a department list with:

  • Status: Complete or Incomplete
  • Action:
    • Start (for incomplete departments)
    • Review (for complete departments)

Select the department you want to work on first.

Tip. Start with the largest academic departments first (Math/Science/English) since they drive most of the section volume.
Replace section-review-department-list.mp4 with your real filename.
2.2 — Inside a department

When you open a department (e.g., Applied Technology Department), key UI elements include:

  • Supervisor chip:
    • “No Supervisor Assigned” with Add a Supervisor (optional)
  • View dropdown: Course View / Teacher View
  • Layout toggle: Grid / Row
  • Menu (three dots) with actions like Rerun Auto Sectioning
  • Search: “Search for teachers or courses”
  • Print button

Most schools work in Course View first to validate demand vs supply.

Replace section-review-inside-department.mp4 with your real filename.
2.3 — Course View cards (demand vs supply)

Each course card typically shows:

  • Course name + code
  • Demand: requests (and sometimes alternates)
  • Supply: # of section(s)
  • Current Est. Class Sizes box

How to interpret:

  • High requests + low sections → consider adding sections
  • Estimated class sizes help you catch overload before scheduling
Tip. Check overloads against Course Caps and room capacities configured in Edit Constraints.
Replace section-review-course-cards.mp4 with your real filename.
2.4 — Add Section (manual)
  1. Click Add Section under a course card.
  2. A To Be Added panel appears with:
    • Course (pre-filled)
    • Teacher(s) fields (blank initially)
    • Add Another Teacher (for co-teaching)
    • Cancel (discard)
    • Save (creates the section)
  3. Click Save to create the new section draft.

Existing section card icons often include:

  • Edit (pencil)
  • Duplicate (two squares)
  • Delete (trash)
  • Add Subsection (if supported by your model)
Replace section-review-add-section.mp4 with your real filename.
2.5 — Rerun Auto Sectioning

From the department menu (three dots), select Rerun Auto Sectioning to trigger recalculation of sections.

While running, the UI may show messages like:

  • “Sections are currently being created…”
  • “We are currently calculating and assigning sections…”

Use this after updating caps, demand assumptions, or course-teacher eligibility.

Good practice. After rerunning, re-check estimated class sizes to confirm overloads were resolved.

Create Labels Create teams/labels and apply them to sections
In Progress

Create teams and labels (custom group tags) and apply them to sections. This is useful for grouping rules that aren’t captured by course codes (e.g., program cohorts, bilingual groups, academy tracks).

Top panels
  • Teams panel with Add Team
  • Labels panel with Add Label
  • Existing labels show a colored dot + label name + an X to delete the label globally
Create a label
  1. Click Add Label.
  2. Enter the label name in the editable row.
  3. Click ✓ to save (or X to cancel).
Apply teams/labels to sections
  • Scroll to sections grouped by department.
  • Each section card shows Course + Teacher + applied tags (dot + name + X to remove).
  • Click Add Team/Label on a section card to attach tags.
Note. If you see a dot with no text, it usually indicates an incomplete/unnamed tag or a tag removed globally. Remove it with X and re-add properly.

Student Labels Assign teams/labels to students for grouping
In Progress

Assign the teams/labels you created to specific students. This starts grouping/teaming logic the scheduler can use later.

4.1 — View: By Team/Label (default)

The page is organized into two columns:

  • Teams (blue header)
  • Labels (orange header)

Each card shows:

  • Team/Label context + associated course/section title
  • Student count (e.g., “0 student(s)”)
  • Placeholder: “No students added yet…”
  • Add Student action

Add a student:

  1. Click Add Student.
  2. Use the search field: Search for a student.
  3. Select the student; they appear in the card and the count updates.
Note. If you see a card title like null - <Course Name>, the label/team name may be missing upstream. Fix labels in Create Labels, then re-apply.
Replace student-labels-by-team.mp4 with your real filename.
4.2 — View: By Student (audit one student)

This view is useful when you want to audit one student’s grouping.

How it works:

  1. Switch the view dropdown to By Student.
  2. Select a student.
  3. Assign teams/labels to that student as needed.
