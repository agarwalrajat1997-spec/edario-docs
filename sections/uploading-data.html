This checklist depends on your SIS choice. Only one flow will be visible below.

2.1 — Excel files to request from your SIS

Contact your SIS support/rep and request the following Excel exports.

The order of the columns in these files is NOT important.
Course Data
  • Course Code
  • Course Name
  • Credits
  • Department
  • Elective/Non-elective
  • Course Length (Full Year, Semester, Quarter)
Last Year’s Sections
  • Semester
  • Course
  • Course Name
  • Room(s)
  • Teacher(s)
  • Credits
  • Beginning Seats
Student Data
  • Student ID
  • Next Year Grade Level
  • Last Name
  • First Name
  • Gender
  • Has Active IEP
Student Requests
  • Student ID
  • School Code
  • Course Code
  • Course Name
  • Alternate Course Code (only if students select alternate courses for their electives)
  • Next Grade Level
Pro Tip. Ask your SIS rep to include stable IDs and to export one school at a time.
2.2 — Quick checklist before uploading

Before you upload into Edario, quickly confirm these items:

  • Each file contains the fields listed in Step 2.1 (extra columns are okay).
  • IDs are present and consistent (Student ID, Course Code, School Code as applicable).
  • Headers are not renamed unless your mapping step expects it.
  • Files are saved as .xlsx or .csv (either is fine).

If something looks wrong, fix the export at the source and re-export rather than editing row-by-row.
Pro Tip. Upload core data first, then validate before moving to the next file.

How to upload Export → upload → map → validate
In Progress

Follow the mini wizard for your SIS. Keep the order: export → upload → validate.

3.1 — Log in to Genesis

Sign in to Genesis using your district credentials.

If you have multiple schools, confirm you’re in the correct school instance before exporting.
Pro Tip. Use a single browser window to avoid exporting from the wrong school/year.
3.2 — Choose the correct school year

Before exporting, change the year dropdown in the top left to the year you want to schedule.

For last year sections, you’ll switch to last year’s school year later in that flow.
Pro Tip. Write down the year you selected so you don’t mix exports across years.
3.3 — Export Courses (navigate to Curriculum)

Click the Scheduling tab, then click Curriculum.

This is where you can pull the full course list and export it to Excel.
Pro Tip. If you don’t see Curriculum, confirm your account has scheduling permissions.
3.4 — Export Courses (search & download Excel)

Without typing any search terms, click Search to load all courses.

Then click the Excel icon in the top-right to download the file.
Pro Tip. Save your export files in one folder (e.g., Genesis_Exports_2026) to stay organized.
3.5 — Check Course file columns

Open the downloaded Course Excel file.

The order of the columns is NOT important, but confirm you have: Code, Description, Credits, Type, Department, Course Length.

Any columns labeled as -- are not important (you can keep them).
Pro Tip. If columns are missing, re-export—don’t manually recreate data unless absolutely necessary.
3.6 — Export Last Year’s Sections

Switch the year dropdown to last year.

Click Scheduling → Sections, then click Search to load all sections.

Click the Excel icon to download the export.
Pro Tip. Sections often include multiple rooms/teachers—export them as-is.
3.7 — Check Sections file columns

Open the Sections Excel file and confirm the required data exists.

The order of the columns is NOT important, but confirm you have: Semester, Course, Course Description, Rooms, Teachers, Credits, Beginning Seats.

Any columns labeled as -- are not important (you can keep them).
Pro Tip. If your export has blanks for Rooms or Teachers, check if those are stored elsewhere in your Genesis configuration.
3.8 — Export Student Requests (Report Writer)

Go to Next Year Scheduling → Reports.

Create a Report Writer Report using the StudentRequest template.

Add columns: courseDescription, alternateCourseCode, nextGradeLevel.

Run the report with output format XLSX, then download the file.
Pro Tip. Save field changes so you can re-run this export without rebuilding the report.
3.9 — Check Student Requests file columns

Open the downloaded Student Requests Excel file.

The order of the columns is NOT important, but confirm you have: Student ID, School Code, Course Code, Course Description, Alternate Course Code, Next Grade Level.

Any columns labeled as -- are not important (you can keep them).
Pro Tip. Student ID + Course Code are the most important keys for matching requests.
3.10 — Export Students (Report Writer + Filters)

Go to Next Year Scheduling → Reports and create a Report Writer Report using the Student template.

Remove schoolCode. Add fields: nextSchoolCode, nextGradeLevel, lastName, firstName, gender, hasActiveIEP.

Add a filter on Next School Code using your SCHOOL_CODE macro, then output as XLSX and download.
Pro Tip. Filtering prevents pulling students who are not returning next year.
3.11 — Check Students file columns

Open the downloaded Students Excel file and confirm the required fields exist.

The order of the columns is NOT important, but confirm you have: Student ID, Next School Code, Next Grade Level, Last Name, First Name, Gender, Has Active IEP.
Pro Tip. If Enrollment Status is included, keep it—it can help troubleshoot missing students later.
3.12 — Upload files in Edario (recommended order)

Upload in this order: Students → Courses → Sections → Student Requests.

After each upload, scan the preview for missing IDs, blank columns, or unexpected duplicates.
Pro Tip. Fix errors immediately; don’t continue uploads on broken base data.
3.13 — Validate & resolve issues

Run validation after uploads. Resolve missing teacher links, missing course codes, or unmatched rosters.

When validation passes, you’re ready to proceed with scheduling.
Pro Tip. Most issues come from IDs or mismatched course/section keys.
3.1 — Contact SIS support

Reach out to your SIS rep/support and request exports for Edario.

Ask for the exports as CSV/Excel. The order of columns is not important.
Support details
 
Email: support@your-sis.com
Phone: +1 (000) 000-0000
Pro Tip. If you don’t have a direct rep, ask your district SIS admin for the correct export templates.
3.2 — Request Course Data export

Ask for a CSV/Excel file containing all course definitions.

Column order is NOT important, but the file should include:

  • Course Code
  • Course Name
  • Credits
  • Department
  • Elective/Non-elective
  • Course Length (Full Year, Semester, Quarter)
Pro Tip. Keep headers stable year-to-year so mapping becomes a one-time setup.
3.3 — Upload Course Data into Edario

Once you’ve confirmed the export includes the required columns, upload it into Edario.

Drag-and-drop the file, or click the upload icon and select your saved CSV/Excel.
Pro Tip. Don’t rename headers unless your mapping step expects it.
3.4 — Request Last Year’s Sections export

Ask for a CSV/Excel file containing the sections from the previous year.

Column order is NOT important, but the file should include:

  • Semester
  • Course
  • Course Name
  • Room(s)
  • Teacher(s)
  • Credits
  • Beginning Seats
Pro Tip. Sections often contain multiple rooms/teachers—export those fields as-is.
3.5 — Upload Last Year’s Sections into Edario

Upload the sections export into Edario using drag-and-drop or the upload icon.

Confirm that identifiers (Course/Section fields) are consistent with your course data.
Pro Tip. If something doesn’t match, fix the CSV and re-upload rather than editing row-by-row.
3.6 — Request Student Data export

Ask for a CSV/Excel file containing student data for next year (including incoming students).

Column order is NOT important, but the file should include:

  • Student ID
  • Next Year Grade Level
  • Last Name
  • First Name
  • Gender
  • Has Active IEP
Pro Tip. Student ID must be stable—it’s the primary key used for matching.
3.7 — Upload Student Data into Edario

Upload the student data export into Edario using drag-and-drop or the upload icon.

Confirm key fields are recognized correctly (especially Student ID and Grade Level).
Pro Tip. Keep one “master” student template per year and append updates over time.
3.8 — Request Student Requests export

Ask for a CSV/Excel file containing student requests for this year.

Column order is NOT important, but the file should include:

  • Student ID
  • School Code
  • Course Code
  • Course Name
  • Alternate Course Code (only if students select alternate courses for their electives)
  • Next Grade Level
Pro Tip. Requests typically drive elective placement—don’t skip alternate course fields if your school uses them.
3.9 — Upload Student Requests into Edario

Upload the student requests export into Edario.

If mapping appears, match the request fields to Edario’s expected columns and save the mapping.
Pro Tip. Mapping is usually a one-time setup once headers are standardized.
3.10 — Validate & finalize

Run validation and resolve missing matches (teacher links, course/section keys, roster mismatches).

When the validation checks pass, you’re ready for scheduling workflows.
Pro Tip. If something looks wrong, fix the CSV and re-upload rather than editing row-by-row.
