Overview Manually place sections, then generate and review
In Progress

Place Sections is where you manually place (some or all) sections into period slots before Edario auto-places anything remaining, then you review issues and generate the schedule.

Step 7 sub-pages. Under the Step 7 header in the left sidebar, you will see: Placement Permissions, Manual Placements, and Placement Review. Each sub-page shows a green check when complete.
Goal. Decide who can manually place sections → optionally drag-and-drop key placements → run Placement Review to generate the schedule. NOTE: generating in Step 7 places sections only; students are placed in Step 8.
Recommended workflow
  1. Placement Permissions: allow department supervisors (optional).
  2. Manual Placements: place priority sections (anchors), leave the rest for auto-placement.
  3. Placement Review: confirm readiness, generate schedule, and review conflicts/warnings.

Placement Permissions Choose which supervisors can manually place sections
In Progress

Placement Permissions decides which department supervisors (if any) are allowed to place sections manually. Any supervisor granted permission gets a chance to place sections before Edario auto-places the rest.

What you’ll see
  • Page title: Placement Permissions
  • Instructional subtitle explaining supervisor placement happens before auto-placement
  • A central table with 3 columns:
    • Department (e.g., 530, Applied Technology, Art, Business, Duties, Electives, etc.)
    • Supervisor(s) pill:
      • a selected supervisor (example: Admin, Edario), or
      • a grey pill: No Supervisor
    • Allow to manually place their sections? (toggle switch per department)
Toggle states (per department)
  • OFF (grey toggle): supervisor cannot do Manual Placements for that department.
  • ON (blue toggle): supervisor can do Manual Placements for that department.

If a department shows No Supervisor, turning permission on has no practical effect until a supervisor exists for that department.

Best practice. Only enable manual placement for departments where you want department-level ownership of anchors (e.g., elective labs, special equipment rooms, or unique rotation patterns).

Manual Placements Drag-and-drop sections into period slots
In Progress

Manual Placements is the interactive placement workspace. You can drag-and-drop section cards into period slots. Any sections not manually placed will be auto-placed by Edario afterwards.

3.1 — Department selection screen

At the top you will see:

  • Title: Manual Placements
  • Instruction: “Any sections that are not manually placed will be auto-placed by Edario afterwards!”

The main card shows: Choose a department to work on:

  • Department name (e.g., 530, Applied Technology, Art, Business, Duties, Electives…)
  • Status: Locked (lock icon), Incomplete, or Complete (green check)
  • Action button:
    • Start (often grey/disabled-looking for incomplete)
    • Review (green) for completed departments
Tip. Start with the department that has the most critical anchors (labs, workshops, shared spaces) to reduce conflicts later.
Replace manual-placements-department-list.mp4 with your real filename.
3.2 — Locked departments (gating behavior)

Some departments show Locked. When locked, the row displays:

  • Message: “This screen is locked because there is a previously unfinished screen.”
  • Link: Go to Last Unfinished Step

This prevents manual placement until earlier required work is finished elsewhere in the wizard.

Good practice. Resolve locked blockers first (usually missing approvals or incomplete earlier screens) before trying to manually place anchors.

3.3 — Inside a department (board header & controls)

When you open a department, you enter a large board view with:

  • Back arrow (returns to department list)
  • Supervisor chip: No Supervisor Assigned (pill) + link Add a Supervisor
  • Label: Department
  • Title: <Department> Schedule (e.g., Applied Technology Schedule)
  • Search bar:
    • Teacher View placeholder: Search for a course or teacher…
    • Classroom View placeholder: Search for a course or classroom…
    • A small filter/settings icon at the right edge (sliders)
  • Right-side counters:
    • Conflicts with a number (e.g., 79, 88)
    • Warnings with a number (e.g., 72)
    • Link: See all
  • View dropdown: Teacher View / Classroom View
Tip. Use Conflicts/Warnings counters as a quick health check after moving anchors. If numbers jump unexpectedly, click See all to investigate.

3.6 — Section cards (draggable items)

Section cards appear inside grid cells in both Teacher View and Classroom View.

What is visible on each card:

  • Dark top bar with:
    • Lock icon (left)
    • Three-dot menu (right)
  • Course name (often truncated, e.g., “Wood Techn…”)
  • Quarter strip: Q1 / Q2 / Q3 / Q4 (some quarters highlighted)
  • Day cycle line: Day(s) 12345 (example shown in UI)
  • Bottom row info:
    • Room indicator + room number (e.g., 502 / 505 / 509 when placed)
    • Small icon cluster at bottom-right with a number (e.g., 15, 20)

Editable / interactive behavior (as shown in your screens):

  • Cards can be grabbed and moved between period cells (drag-and-drop placement).
  • Empty cells show a faint + icon indicating a valid drop/add target.
  • Red-highlighted cards/cells indicate a conflict/invalid placement or a flagged state during moves.
  • The ellipsis menu suggests additional card actions (depends on your app configuration).
Best practice. Manually place only the hardest-to-solve anchors first (special rooms, shared staff, multi-period blocks), then let auto-placement fill the rest.

Placement Review Generate schedule and review conflicts/warnings
In Progress

Placement Review is the final check + schedule generation step for placing sections. You will confirm readiness, generate the schedule, then review conflicts/warnings in the generated board.

4.1 — Department list with lock states

You may see a department list similar to Manual Placements:

  • Department name
  • Status:
    • Locked (lock icon) with message: “This screen is locked because there is a previously unfinished screen.”
    • Complete (green check)
  • For locked rows:
    • Link: Go to Last Unfinished Step
  • For complete rows:
    • Review button (green)

This reinforces that review depends on completing required earlier work.

Replace placement-review-locked-list.mp4 with your real filename.
4.2 — “Whenever You are Ready!” (Generate Schedule gate)

A centered confirmation card appears with:

  • Illustrated calendar icon with a green check
  • Heading: Whenever You are Ready!
  • Explanation text confirming constraints, approvals, and manual placements are done
  • NOTE: “This will only place sections. No students will be placed until the next step.”
  • Main button: Generate Schedule (large green CTA)

4.4 — Generated schedule view (post-generation)

After generation, you see a schedule board similar to Manual Placements, showing:

  • Department label + schedule title (e.g., Health and Physical Education Schedule)
  • Search bar + view dropdown (Teacher View / Classroom View)
  • Conflicts / Warnings counters (with See all)
  • Period grid across the top (including combined blocks like Period 4/5)
  • Teacher rows on the left
  • Section cards placed across periods

Issue highlighting:

  • A card may appear red-highlighted to indicate a conflict/issue needing review.
  • Use Conflicts/Warnings and See all to locate the cause and decide if anchors need adjustment.
Done standard for Step 7. Placement Permissions completed → Manual Placements done (or skipped) without locked blockers → Placement Review reaches “Whenever You are Ready!” and you generate the schedule → conflicts/warnings reviewed.
