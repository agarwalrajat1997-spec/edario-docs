In Edario, classrooms are not just room names. A classroom encodes capacity, type, department alignment, and both hard and soft compatibility signals that influence where sections can (and should) be placed.

Where to find it

School Data → Classrooms

Use this area to keep room attributes consistent and to avoid common placement failures caused by capacity/type mismatches.

Best practice: Keep room capacity accurate and classify special rooms using Classroom Type (labs, gyms, studios). This makes later placement and reports much more reliable.

Use Add New Classroom to create a new room, or open an existing room to edit its attributes. This setup controls how the scheduler may assign sections to rooms.

2.1 — Classroom Name

A text input field labeled Classroom Name.

  • Use the official room identifier (e.g., 110, Gym, Lab 201).
  • Keep naming consistent with your SIS / facility labeling.

Consistent names improve search, reporting, and reduce accidental duplicates.

Replace data-classrooms-name.mp4 with your real filename.
2.2 — Departments (multi-select chips)

A section titled Departments with pill-style chips you can toggle on/off (examples: Applied Technology, Art, Business, Science, Special Education).

What this does:

  • Associates the room to one or more departments.
  • Helps filtering and ownership views across the app.
  • Supports department-based workflows later (constraints, manual placement, reporting).

Recommended: assign every classroom to the department(s) that actually use/own it.

Recommended GIF/video: “Assign a classroom to departments via chips.” Replace data-classrooms-departments-chips.mp4.
2.3 — Classroom Type (category chips)

A section titled Classroom Type with selectable type chips (examples: Art, Auditorium, Band, Cafeteria, Ceramics, Choir Room, Computer Lab, Gym, Lab - Chem, Lab - Physics, Lecture Hall, etc.).

Why this is powerful:

  • Improves room filtering in placement views and reports
  • Supports compatibility logic (lab courses prefer lab rooms)
  • Helps staff reason about placement quickly

Choose the type that best describes the room’s real-world function.

Example: Set Classroom Type = Lab - Chem for chemistry lab rooms so lab courses can be restricted correctly.
Recommended GIF/video: “Pick Classroom Type = Lab - Chem.” Replace data-classrooms-type-lab-chem.mp4.
2.4 — Max Classroom Capacity

A numeric field for Max Classroom Capacity (example: 30).

What this enforces:

  • Hard limit: sections assigned to this room should not exceed this capacity.
  • Improves seat feasibility and prevents unsafe/invalid placements.

Keep this accurate; capacity is one of the most common sources of schedule issues.

Replace data-classrooms-capacity.mp4 with your real filename.
2.5 — Allows multiple sections? (Yes/No toggle)

A labeled toggle: Allows multiple sections?

This is critical for spaces like gyms, auditoriums, and cafeterias that can host more than one section in the same period.

Guidance shown (example):

  • Gym can hold multiple sections in same period.

Use carefully:

  • Set to Yes only if the space truly supports concurrent sections.
  • Keep as No for normal classrooms and labs.
Recommended GIF/video: “Set Allows multiple sections = Yes (Gym example).” Replace data-classrooms-multi-sections-gym.mp4.
2.6 — Course(s) that NEED to be in this classroom (hard binding)

Section title:

Course(s) that NEED to be in this classroom

  • Empty state (no courses bound yet)
  • Action button: Add Course that Needs Classroom

This is the reverse of a course-level classroom restriction: instead of “course needs room,” it is “room must host course.”

Use cases:

  • A single TV Studio where TV production must meet
  • A specific workshop with required equipment
  • A dedicated band/choir room for rehearsals

Use this only when the room binding is truly non-negotiable.

Recommended GIF/video: “Bind a course to a specific classroom.” Replace data-classrooms-bind-course-to-room.mp4.
2.7 — Teacher(s) Who Prefer This Classroom (soft preference)

Section title:

Teacher(s) Who Prefer This Classroom

  • Empty state (no preferences set)
  • Action button: Add Teacher who Prefers Classroom

This encodes a preference signal (typically softer than “needs”). It helps Edario choose better room placements when multiple valid rooms exist.

Typical use:

  • Teachers with “home rooms”
  • Teachers tied to a building/wing
  • Preferred lab or studio spaces

Avoid using preference as a hard rule unless your process requires strict room assignment.
